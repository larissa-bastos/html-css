Considerações iniciais

Seu site deve se adaptar a diferentes tamanhos de tela

A gente abre sites hoje em dia em diversos dispositivos diferentes: smartphones, tablets, notebooks, tvs, etc. Com isso em mente, a gente vai ter que adaptar o conteúdo a muitos tamanhos de tela.
A mídia que mais tem problema com isso são as imagens. Uma TV abre imagens enormes, mas se abrirmos o mesmo site num celular, vamos ter que redimensionar a imagem com CSS. O problema é que mudar com CSS não muda o tamanho do arquivo de imagens, o que aumenta o consumo de dados e deixa o site pesado e lento. Sites assim fazem as pessoas cansarem e desistirem logo, fechando o site, diminuindo seu valor no SEO.

Imagens flexíveis

Então para fazer tudo da melhor forma, a gente quer gerar imagens de vários tamanhos e fazer o carregador carregar a certa para cada dispositivo que o acessar. As tags são picture e source.

Imagens dinâmicas

Primeiro: tamanho não é documento. Grandes imagens formam um site pesado e afeta o SEO. Lembrando de como colocar uma imagem: img src="link da sua imagem" alt "texto alternativo para leitores de tela".
Mas então o que fazemos para adaptar uma imagem a diferentes dispositivos e não ficar faltando nada? A gente cria várias imagens de diferentes tamanhos e avisa pra o site qual a gente quer que apareça dependendo do tamanho do dispositivo. Para testar, vamos aumentar e diminuir o zoom do browser. NOVA TAG: picture. Dentro dela fica a img e outra tag, acima da img, chamada SOURCE.
